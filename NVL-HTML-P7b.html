TÉRMINOS CON A

  • a:

    Es el elemento que nos permite enlazar nuestros documentos HTML entre sí, o a otras webs externas a la nuestra.

  • address:

    Muestra la información de contacto de una persona, como por ejemplo, nombre y dirección de correo electrónico, es decir, con este tag, podemos especificar información de contacto relevante sobre la persona, empresa que realiza una web, pero su utilización no está pensada para una dirección física de un lugar, sino más bien para la datos de contacto online.

  • area:

    Con este elemento podemos considerar una imagen como una serie de porciones. Por ejemplo en el caso de un mapa podemos definir diferentes partes de la imagen como áreas del mismo para establecer ubicaciones de lugares mediante coordenadas.

  • article:

    Este elemento es uno de los nuevos tags de HTML 5 que nos otorga la posibilidad de dar un significado a ciertos contenidos o grupos de contenidos en la web, como por ejemplo, noticias o comentarios.

  • aside:

    Nos permite establecer una relación entre contenidos similares. Cuando vemos un tag de este tipo, sabemos que el contenido que se encuentra en su interior, se relaciona con otro contenido de la misma página.

  • audio:

    El tag audio, nos permite integrar ficheros de sonido de forma facil y sencilla sin necesidad de recurrir a un reproductor flash.

TÉRMINOS CON B

  • bdi:

    El tag permite tratar una parte del texto de forma independiente al resto y darle una dirección diferente. Un uso común de este tag, se da en idiomas como el chino, japonés o el árabe, que emplean un sistema de escritura de texto diferente al occidental.

  • body:

    Este elemento define el cuerpo principal del documento HTML que estamos editando. En HTML 4 este tag tenía la posibilidad de especificar atributos, pero estaban en desuso, mientras que en HTML 5 han sido definitivamente eliminados, por lo que el tag ya no tiene ningún atributo posible.

  • br:

    El elemento br inserta un quiebre de línea, causando que el texto continúe en una línea nueva.

  • button:

    Con este tag, podemos definir botones en la web. Es uno de los elementos mas utilizados en formularios para diversas acciones, como enviar el formulario, acceder a un arbol de directorios, etc. En HTML 5 se han añadido nuevos atributos a este elemento que facilitan su integración con los formularios y nos permiten decirle al navegador de forma mas sencilla las diferentes acciones que se llevan a cabo cuando se pulsa el botón.

TÉRMINOS CON C

  • canvas:

    El elemento canvas, nos permite crear un contenedor en el que podemos dibujar cualquier tipo de gráfico, para lo cual se deberá utilizar algún lenguaje de script, como por ejejmplo Javascript para definir como se dibujan los gráficos y su comportamiento.

  • caption:

    Se utiliza para agregar un título general a una tabla sin necesidad de recurrir al tag y los estilos, de tal forma que se puede añadir un titulo sin que se corresponda con el formato del resto de la tabla.

  • code:

    El tag code, forma parte de un conjunto de elementos que nos permiten establecer el resaltado de texto como por ejemplo, el de un editor de programación en un documento HTML. En concreto, todo el texto que vaya dentro de una etiqueta code, será considerado como parte del código de un programa.

  • colgroup:

    Con este elemento podemos crear una o varias columnas en una tabla. En HTML 5 solo se puede utilizar el atributo span (permite establecer relleno a las columnas de la tabla), ya que el resto han sido eliminados.

  • command:

    Este elemento nos permite crear botones de distintos tipos como parte de un menú desplegable, o bien para definir un atajo por combinación de teclas en cualquier lugar de nuestra web.

TÉRMINOS CON D

  • datalist:

    El elemento datalist representa una lista de opciones predefinidas que pueden ser utilizadas para proveer sugerencias en muchos controles de formularios. Estas opciones pueden ser provistas por el navegador en una lista o como una referencia de autocompletado, dependiendo del navegador y el control con el cual este elemento está relacionado. Un datalist nunca es representado por sí mismo.

  • details:

    Con este tag, podemos añadir información que se considerará adicional a la ya existente y que podrá ocultarse o hacerse visible bajo petición del usuario. Tiene un atributo open, que si esta presente, la información que haya dentro de este tag, quedará desplegada por defecto. Este elemento sólo lo interpreta correctamente Google Chrome. En caso de cualquier otro navegador se interpreta como texto normal.

  • dfn:

    Pertenece al conjunto de elementos que permiten realizar resaltado de texto. Todo el contenido que vaya en esta etiqueta, se considerará la definición de una palabra. Esto es útil a la hora de crear un diccionario o un glosario de terminos.

  • div:

    Es el elemnto más utilizado en HTML 4 y sirve para agrupar contenidos. Esto nos permite dar estilo de forma sencilla a una sección o grupo de elementos. El tag div suele utilizarse para formar la estructura básica (posición de los diferentes menús, imágenes, etc). En HTML 5 disponemos del tag section que nos permite dar un significado a las diferentes partes de la página.

TÉRMINOS CON E

  • em:

    Dentro de los elementos para el resaltado de texto, es aquél que nos permite marcar un texto con una mayor importancia, pero desde el punto de vista semántico y no desde visual.

  • embed:

    Dentro de este elemento podemos insertar contenido externo a la web, como flash, java, etc.

TÉRMINOS CON F

  • fieldset:

    Es el tag que nos permite agrupar diferentes campos de un formulario que estén relacionados entre sí, como por ejemplo, los datos personales en un formulario de registro de usuario.

  • figcaption:

    Es un tag complementario a figure que nos permite asignar un título a la imagen que va dentro de este elemento. Podemos situarlo sobre la imagen o bajo ella.

  • figure:

    El tag figure nos permite delimitar un contenedor para imágenes, gráficas, etc. si se complementa con el tag figcaption se le puede añadir un título a la imágen o grafica que se muestre. Un uso habitual de este elemento se da en el caso en que queremos englobar una imágen con título por ejemplo.

  • footer:

    Nos permite englobar el contenido que corresponde al pie de nuestra pagina web. Por ejemplo la información de contacto de nuestra empresa.

  • form:

    Es el tag que se utiliza para definir un formulario.

TÉRMINOS CON H

  • h1, h2, h3, h4, h5, h6:

    Estos elementos nos permiten crear títulos de diferentes tamaños donde h1 es el tamaño mayor, y h6 es el mas pequeño, pudiendo definir títulos generales, títulos de sección, etc.

  • head:

    Este elemento, define la cabecera de nuestro documento HTML, donde podemos hacer referencias a todos los scripts, hojas de estilos, titulo del documento, datos para buscadores, etc.

  • header:

    El tag header nos permite definir contenidos como preámbulo, que es muy útil en casos como una noticia, un documento de texto o un artítulo periodístico, así como una entrada de blog. Nos permite insertar texto que describa brevemente el contenido sobre el que va a tratar dicho documento, noticia o entrada. En definitiva, nos permite que un contenido determinado se pueda diferenciar entre la cabecera, que se definiría con este tag y el cuerpo, que por ejemplo se podría definir con un tag article.

  • hgroup:

    Es un tag contenedor que nos permite agrupar elementos de título como h1, h2, etc. Se puede emplear por ejemplo para crear un índice dentro de la web, ya que todos los elemnetos son títulos de mayor o menor importancia.

  • hr:

    Este elemento nos permite insertar una linea horizontal que establece una división en el texto.

  • html:

    Es el tag que define el comienzo y fin de un documento.

TÉRMINOS CON I

  • i:

    Esta etiqueta HTML ha sido utilizada comúnmente para hacer que un texto concreto pase a la llamada letra cursiva o itálica, pero ahora, se emplean las hojas de estilos para formatear las fuentes y el tag i, ha quedado relegado a un uso en caso de que no exita otro que represente mejor lo que queremos definir o para cuando se quiere destacar un tecnicismo o un nombre.

  • iframe:

    Este tag es un contenedor que nos permite cargar otra página web en una parte de la nuestra. Esta etiqueta conlleva ciertos riesgos de seguridad, ya que estamos cargando contenido ajeno al nuestro que no podemos controlar.

  • img:

    Permite la inserción de imágenes de múltiples formatos en la web, como jpg, gif, png, etc. Este tag a penas tiene cambios con respecto a HTML 4. Únicamente, han sido eliminados aquellos atributos que se habían quedado obsoletos en favor de las hojas de estilos como los referentes a al alineación o los bordes.

  • input:

    Este elemento se utiliza para definir un campo dentro de un formulario. Se pueden definir diversos tipos de campo con este tag, como botones de envío, campos de texto, checkboxes, etc.

TÉRMINOS CON K

  • kbd:

    Forma parte del conjunto de etiquetas que permiten definir el resaltado de texto. Este tag nos permite tratar un texto como si fuese una entrada de teclado de un programa.

  • keygen:

    Este tag se utiliza para el cifrado a la hora de enviar un formulario, creando un generador de claves para el mismo. Este campo nos permite utilizar algoritmos de cifrado como el rsa, dsa o ec para el cifrado del formulario, empleando un sistema de envío de clave pública.

TÉRMINOS CON L

  • label:

    Este elemento define una etiqueta para un campo del formulario, de forma que nos permite denominar ese campo de alguna forma, es decir, nos permite dar un nombre a un campo del formulario, el cual nos servirá tambien para referirnos a ese campo de tal manera que si se selecciona con el ratón ese texto se interprete como un focus en el campo en cuestión. Este tag simula una etiqueta que se asigna al campo como un texto descriptivo del mismo.

  • legend:

    Este elemento nos permite agregar un título que nos puede servir para ofrecer una información adicional sobre un grupo de campos de un formulario fieldset de esta manera podemos por ejemplo agrupar los campos referentes a los datos personales de un usuario que se registra en nuestra web (nombre, apellidos, dirección, etc) con un título que indique: “Datos Personales”. De esta forma queda claramente diferenciado del resto de campos que contenga el formulario de registro.

  • li:

    El tag li define un elemento dentro de una lista ya sea ordenada o desordenada ul. En HTML 5 los atributos type y value han sido eliminados. Por ejemplo, en un índice, los tags li son los elementos que albergan el contenido del índice.

  • link:

    Este elemento permite relacionar el HTML de nuestra web con el css. También admite otros tipos de documento, pero lo más común, es enlazar las hojas de estilos con las plantillas HTML, de esta forma, podemos definir los estilos de la web sin necesidad de hacerlo directamente en la plantilla, que es incomodo, además de costoso y se duplica muchísima información.

TÉRMINOS CON M

  • map:

    Es un tag que permite dividir una imagen en secciones que llevarán a un enlace que aportará mas información. Por ejemplo en un mapa del mundo, podemos establecer secciones a través de coordenadas que definan cada uno de los continentes, donde cada continente tiene un enlace que nos lleve a un mapa con todos los países que alberga ese continente.

  • mark:

    Es el elemento que nos permite marcar una parte del texto con un color de subrayado emulando a los rotuladores de marcado que se utilizan sobre papel.

  • menu:

    El tag menu se emplea para definir listados de opciones desplegables, al igual que los listados de opciones que se despliegan cuando hacemos click con el botón derecho del ratón en en un archivo en el sistema operativo. Cabe destacar que este tag no funciona en ningún navegador en la actualidad de los que se utilizan habitualmente.

  • meta:

    Este tag siempre se encuentra como parte de la información que proporciona la cabecera HTML (tag head) nos proporciona la información necesaria para indexar nuestra página web con los buscadores y facilitar su localización.

  • meter:

    Es un tag nuevo de HTML 5 que nos permite establecer medidas para un rango que tiene un valor máximo conocido, como puede ser por ejemplo, la votación por los diferentes candidatos, ya que existe un número máximo de votos posibles que es conocido.

TÉRMINOS CON N

  • nav:

    Define una parte de la web como un menú de navegación, es decir, dentro de este tag, se agrupa una serie de enlaces que generalmetne suele ser el menú de secciones de la web, también conocido como menú de navegación.

  • noscript:

    Este elemento tiene su principal utilidad para ofrecer una alternativa para aquellos usuarios que tengan un navegador que no soporte lenguajes como JavaScript. Esto permite mostrar contenido en dichos casos y también evita que en caso de que no se soporten los lenguajes de script, se muestre el código del script en la pantalla lo que supone un riesgo de seguridad importante.

TÉRMINOS CON O

  • object:

    Es el tag que nos permite insertar código de otros medios externos como pueden ser flash, audio, video, aplicaciones Java, etc. Tambien se puede utilizar este tag en lugar de iframe para insertar otra web como parte de la que se está ejecutando.

  • ol:

    Con este tag, podemos definir listas ordenadas a modo de esquemas numerados cn una determinada notación.

  • option:

    Este es el tag que se utiliza en los combos select para definir las diferentes opciones del listado. En HTML 5, también se usa en el tag datalist.

  • output:

    A través de este tag, podemos mostrar el resultado de una operación que realice un script, como puede ser un calculo de cambio de moneda, una ecuación matemática o una conversión de unidades de medida.

TÉRMINOS CON P

  • p:

    Este es un tag de tratamiento de texto que nos permite crear parrafos en HTML. Además, por defecto, los tags p, provocan un salto de linea forzado al final y al principio del párrafo. En HTML 5, la alineación de párrafo se está permitida, pero lo más recomendable es el uso de css para este fin.

  • param:

    Es un tag que se utiliza como complemento del tag object para comunicarnos con la palicación externa que contiene, mediante el paso de parámetros Los atributos valuetype y type han sido eliminados en HTML 5.

  • pre:

    Este elemento nos permite insertar texto formateado en HTML, es decir, todo el texto que se introduzca dentro de este tag, se mostrará tal y como lo hemos escrito en el documento HTML, respetando los saltos de linea, espacios y demás características del texto.

  • progress:

    Es el tag que se emplea en lugar del javascript para representar el progreso de una tarea, como por ejemplo la copia de un archivo, una descarga, un test de tiempo de respuesta, etc.

TÉRMINOS CON R

  • rp:

    Este elemento se usa junto con ruby y rt para mostrar la pronunciación de los caracteres orientales. Concretamente, el tag rp indica lo que mostrara el navegador en caso de que éste no funcione correctamente con el tag ruby

  • rt:

    Este tag está pensado para ser utilizado junto al tag ruby como complemento descriptivo de la pronuciación de los caracteres del continente asiático.

  • ruby:

    Se emplea junto con los tags rt y rp para escribir caracteres orientales. Por ejemplo, es útil para una definición de un carácter asiático dentro de un diccionario con su pronunciación.

TÉRMINOS CON S

  • s:

    Este tag hace referencia a texto incorrecto. Por ejemplo un uso que se le puede dar a este elemento es el de la corrección de exámenes online, para que el alumno sepa que partes del mismo no son correctas y que partes sí lo son. El efecto visual es el mismo en HTML 5 que en HTML 4, pero mientras que en HTML 4 se le daba un uso para tachar partes del texto, en el caso de HTML 5, tiene su utilidad para el texto que no es correcto dentro de un contexto concreto, es más una redefinición semántica que de funcionalidad, ya que el efecto es el mismo.

  • script:

    Es el elemento que nos permite ejecutar código de lenguajes de programación en el lado del cliente pudiendo cargar animaciones, y en general, contenido que cambia dinámicamente en función de diversas condiciones. El más habitual es el Javascript.

  • section:

    Es un elemento con un uso similar a div de HTML 4, que nos permite agrupar contenido relacionado entre si. Por ejemplo, los comentarios de una noticia, el contenido general de una noticia, etc. La diferencia entre el tag div y el tag section, en principio es meramente semántica, pero hay quien recomienda que los tags section se empleen para definir zonas de contenido relacionado y no para dividir de manera estructural la web.

  • select:

    Se emplea junto con el tag option, para crear listas despelegables de selección de opciones. Tiene múltiples usos para definir infinidad de tipos diferentes de listados, como por ejemplo, las diferentes profesiones que puede tener una persona, la edad de una persona, el sexo de una persona, etc.

  • source:

    Es un tag complementario que se utiliza con los elementos video y audio cuando se pretende que nuestro fichero tenga múltiples réplicas que permitan una mayor disponibilidad del contenido multimedia a reproducir.

  • style:

    Es el elemento que nos permite dar estilo a los documentos HTML de forma directa y sin necesidad de hojas de estilos. Aunque su uso esta desaconsejado, en ciertas ocasiones es necesario para partes muy concretas y específicas de la web.

  • sumary:

    Este elemento se combina con el tag details para crear secciones desplegables dentro de la web. Por ejemplo se puede utilizar para crear un desplegable que contiene el resumen de una noticia o las características de un producto. Se debe tener en cuenta que este tag sólo lo soporta Google Chrome actualmente, por lo que en el resto tendremos que utilizar JavaScript para emular este efecto.

TÉRMINOS CON T

  • table:

    Es uno de los tags para la generación de tablas. Concretamente, este elemento es el que define la tabla en sí. Para definir las filas y columnas de la tabla utilizaremos los tags tr y td.

  • tbody:

    Dentro de los elementos de los que dispone HTML para la generación de tablas, tbody define el cuerpo de la tabla, donde se crearán todas las filas y columnas de dicha tabla que albergarán el contenido de la misma.

  • td:

    Este tag es mediante el cual podemos crear las diferentes columnas de una tabla. Se encuentra siempre dentro de un tag tr.

  • textarea:

    Define un área de texto dentro de un documento HTML. Se utiliza principalmente en formularios para insertar HTML (por ejemplo el código de un iframe), textos sin límite de caracteres, etc.

  • tfoot:

    Este elemento define aquel contenido que formará parte del pie de una tabla HTML.

  • th:

    Similar al tag td, pero para filas que se definen en la cabecera de una tabla. Este tag tambien ha sufrido cambios en cuanto a sus atributos y tan sólo conserva los siguientes: colspan, rowspace y scope.

  • thead:

    Es el elemento que define el contenedor para la todo lo que tenga relación con la cabecera de la tabla. Los atributos referentes a este elemento han sido eliminados.

  • time:

    Es el elemento que se utiliza para gestión de las fechas en HTML 5, por ejemplo para otorgar una fecha de publicación a un articulo o para una fecha de entrega de un proyecto.

  • tr:

    Un tag tr representa una fila de una tabla HTML, permitiendo agrupar varios tags td.

  • track:

    Este tag proporciona el soporte necesario para mostrar información relacionada con un elemento que se está reproduciendo, como por ejemplo un fichero de audio o de vídeo, de tal forma que con este elemento podemos especificar texto informativo del contenido multimedia, como puede ser su título, álbum, artista, subtitulado en un vídeo, título de un capítulo, traducción de efectos de sonido mediante onomatopeyas, etc. Destacar que este tag no funciona en ninguno de los navegadores habituales en la actualidad.

TÉRMINOS CON U

  • u:

    Al igual que el tag s, este tag ha adquirido un significado en HTML 5, de tal forma, que ahora se emplea para subrayar texto que está mal formado o escrito de forma incoherente.

  • ul:

    Este tag representa una lista que no tiene un criterio de orden de esquema. A diferencia de las listas generadas con el tag ol, no siguen ningún criterio de orden preestablecido y su forma de representar los elementos de la lista es más apropiada para casos como por ejemplo, las características de un producto, las ventajas e inconvenientes de una decisión empresarial, etc.

TÉRMINOS CON V

  • var:

    Este tag representa el contenido como una variable de un lenguaje de programación. Pertencece al conjunto de elementos que se emplean para el resaltado de texto.

  • video:

    Hasta el momento los tags object e iframe eran la única herramienta de la que se disponía para insertar video en la web, lo cual nos obligaba a depender, o bien de un servidor de vídeo externo, como Youtube o Vimeo, o bien emplear flash para insertar un reproductor en la web que nos permitiera reproducir el vídeo en nuestro propio servidor. Con el tag video, podemos hacer esto sin necesidad de flash ni de depender de otro sistema externo. Este elemento permite insertar vídeo en un contenedor con los controles de reproducción típicos de play, pause, control de volumen y control de la línea de tiempo de reproducción para poder situar el vídeo en un instante concreto dentro de la linea temporal del mismo.
